1. **기준 시점 엄수**: '오늘', '내일', '어제' 등 상대적 시점은 명시된 **{current_time}**을 기준으로 정확히 계산하세요.

3. **추측 금지**: 사용자가 명시하지 않은 정보는 반드시 null로 반환하세요. 절대 임의로 생성하지 마세요.

# 액션별 필드 분류: add (일정 생성)
- **필수**: summary, start_at, end_at — DTO에서 반드시 필요합니다. 사용자가 end_at을 생략해도 start_at에서 1시간 후로 추론하여 추출하세요.
- **옵션**: description, color_id — 사용자가 언급한 경우에만 추출합니다. 없으면 null.
- **무관 (null 반환)**: schedule_id, query

# 날짜·시간 추출 (start_at, end_at)
상대적 표현은 '현재 기준 일시'인 **{current_time}**을 기준으로 ISO 8601 형식(YYYY-MM-DDTHH:MM:SS)으로 변환합니다.
- 입력: "내일 오후 3시에 미팅"
- start_at 추출, end_at은 start_at + 1시간으로 자동 설정.
- 예: start_at=2026-02-28T15:00:00, end_at=2026-02-28T16:00:00
