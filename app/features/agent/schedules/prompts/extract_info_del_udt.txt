사용자의 대화에서 수정 또는 삭제할 일정의 ID(schedule_id)와 변경할 정보를 추출하세요.

# 추출 원칙
1. **대상 식별 (schedule_id)**: 
   - 사용자가 명시적으로 숫자 ID를 언급하거나(예: "30번"), '최근 검색 결과'의 특정 항목을 지칭하는 경우 해당 `schedule_id`를 추출하세요.
   - '최근 검색 결과'에 항목이 여러 개 있고 사용자가 "첫 번째 거" 등으로 지칭하면 해당 항목의 ID를 추출하세요.
2. **대상 변경 감지 (CRITICAL)**: 
   - '현재 저장된 정보'의 `schedule_id`와 다른 새로운 ID를 언급하거나 다른 대상을 작업하려는 의도를 보인다면, 기존 정보를 무시하고 **새로운 대상에 대한 정보만 추출**하세요.
3. **정보 갱신 (Update 시)**:
   - `action`이 `update`인 경우, 사용자가 변경하려는 필드(summary, start_at, end_at, description, color_id)를 추출하세요.
   - 시간 정보는 **{current_time}**을 기준으로 정확히 계산하여 ISO 8601 형식으로 변환하세요.
4. **최신 메시지 우선**: 전체 대화를 참고하되, 가장 마지막 메시지에서 새롭게 언급된 정보에 집중하세요.
5. **추측 금지**: 사용자가 언급하지 않은 정보는 반드시 null로 반환하세요.

# 액션별 필드 분류: update, delete
- **delete**: 
  - 오직 `schedule_id`만 추출합니다. 
  - 나머지 필드는 모두 null을 반환하세요.
- **update**: 
  - `schedule_id`와 사용자가 변경하고자 하는 필드만 추출합니다. (예: "제목을 회의로 바꿔줘" -> summary="회의")
  - 변경 대상이 아닌 필드는 null을 반환하세요.

# 예시
- 사용자: "30번 일정 삭제해줘"
  - 결과: schedule_id=30
- 사용자: "아까 찾은 것 중에서 2번째 거 지워"
  - 결과: schedule_id=(최근 검색 결과의 2번째 항목 ID)
- 사용자: "123번 제목을 '팀 회의'로 바꿔줘"
  - 결과: schedule_id=123, summary="팀 회의"
