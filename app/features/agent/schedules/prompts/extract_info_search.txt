# 추출 원칙 (중요: 시간 계산 기준)
1. **기준 시점 엄수**: '오늘', '내일', '어제' 등 상대적 시점은 명시된 **{current_time}**을 기준으로 정확히 계산하세요.
2. **Case D: 기간(Period/Range) 조회 규칙**:
   - "N달 후"는 정확히 N개월 뒤 **해당 월 1개월 범위**만 의미합니다.
   - 월말이어도 현재 월(Month) 기준으로 계산하세요.
   - 예시 (기준: 2026-02-28):
     - 이번 달 → start_at=2026-02-01T00:00:00, end_at=2026-02-28T23:59:59
     - 다음 달 → start_at=2026-03-01T00:00:00, end_at=2026-03-31T23:59:59

# 액션별 필드 분류: search (일정 조회)
- **특징**: query 또는 start_at 중 하나 이상이 있어야 의미 있는 검색이 됩니다.
- **관련 필드**: query, start_at, end_at
- **무관 (null 반환)**: summary, description, color_id, schedule_id

# 제목·키워드 추출
- **query**: 조회할 실질적인 검색 키워드.
  - **중요**: "일정", "검색", "찾아줘", "내역" 등은 사용자의 **의도(Action)**이지 검색 대상이 아닙니다. 이러한 단어는 절대 포함하지 마세요.
  - 사용자가 명확한 검색 대상을 언급하지 않고 "이번달 일정 보여줘"라고만 했다면, `query`는 `null`로 추출해야 합니다.
  - 날짜/시간 정보를 포함하지 마세요.
  - 예시:
    - "이번 달 **회의** 찾아줘" -> query="회의"
    - "이번 달 일정 보여줘" -> query=null (기간만 추출)
    - "**휴가** 내역 검색" -> query="휴가"
